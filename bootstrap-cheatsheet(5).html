Einführung in Bootstrap

Was ist Bootstrap?

Bootstrap ist ein kostenfreies, Open-Source Front-End-Framework für die Entwicklung von responsiven Websites und Webanwendungen. Es wurde von Twitter-Entwicklern erschaffen und ist eines der beliebtesten CSS-Frameworks weltweit.

Definition: Ein CSS-Framework ist eine vorgefertigte Sammlung von CSS-Dateien, die grundlegende Stilregeln, Layouts und Komponenten bereitstellen, um die Webentwicklung zu beschleunigen.

Warum Bootstrap verwenden?

  • Zeitsparend - vorgefertigte Komponenten
  • Responsive - Mobile-First Ansatz
  • Konsistentes Design - über verschiedene Browser hinweg
  • Große Community - viele Ressourcen und Hilfestellungen
  • Anpassbar - kann an eigene Bedürfnisse angepasst werden
  • Dokumentation - sehr ausführlich und hilfreich

In der aktuellen Version (Bootstrap 5) wird jQuery nicht mehr benötigt! Alle JavaScript-Funktionalitäten wurden mit Vanilla JavaScript neu geschrieben.

Geschichte von Bootstrap

  • 2011 - Bootstrap 1.0 wird veröffentlicht (von Twitter Entwicklern)
  • 2013 - Bootstrap 3.0 (erster Mobile-First Ansatz)
  • 2018 - Bootstrap 4.0 (Flexbox, Sass statt Less)
  • 2021 - Bootstrap 5.0 (ohne jQuery, verbesserte Grid, RTL Support)
  • 2023 - Bootstrap 5.3 (aktuellste stabile Version)
Installation und Einbindung

Wie füge ich Bootstrap in mein Projekt ein?

Methode 1: CDN (Content Delivery Network)

Am einfachsten: Füge diese Links in deinen HTML-Header ein:

<!-- CSS only -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">

<!-- JavaScript Bundle with Popper -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>

Vorteile CDN: Schneller, Caching-Effekte, kein lokaler Speicherplatz nötig.
Nachteile: Funktioniert nicht offline, Abhängigkeit von externem Dienst.

Methode 2: Download

  1. Gehe auf getbootstrap.com
  2. Lade die kompilierte Version herunter
  3. Entpacke die ZIP-Datei
  4. Binde die CSS- und JS-Dateien in dein Projekt ein:
<!-- CSS -->
<link href="path/to/bootstrap.min.css" rel="stylesheet">

<!-- JavaScript -->
<script src="path/to/bootstrap.bundle.min.js"></script>

Methode 3: Package Manager (npm, yarn)

# Mit npm
npm install bootstrap

# Mit yarn
yarn add bootstrap

Basisstruktur eines Bootstrap-Projekts

<!DOCTYPE html>
<html lang="de">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Meine Bootstrap Seite</title>
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body>
    <!-- Hier kommt der Inhalt -->
    
    <!-- JavaScript am Ende des Body für bessere Performance -->
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>

Die <meta name="viewport"> Zeile ist EXTREM WICHTIG für responsives Design! Ohne diese funktioniert Bootstrap nicht richtig auf mobilen Geräten.

Das Grid System

Das Grid-System - Grundkonzept

Das Grid-System ist das Herzstück von Bootstrap. Es ermöglicht das Erstellen von flexiblen, responsiven Layouts mithilfe eines 12-Spalten-Systems.

Das Grid basiert auf CSS Flexbox und ist in 12 Spalten aufgeteilt. Warum 12? Weil sich 12 gut durch 1, 2, 3, 4 und 6 teilen lässt!

Die drei Hauptkomponenten des Grid-Systems

  1. Container - hält alles zusammen und zentriert
  2. Rows (Zeilen) - horizontale Gruppen von Spalten
  3. Columns (Spalten) - der eigentliche Inhalt

Tooltips

        
Tooltips zeigen zusätzliche Informationen an, wenn der Benutzer mit der Maus über ein Element fährt:

        
        
<button type="button" class="btn btn-secondary" data-bs-toggle="tooltip" data-bs-placement="top" title="Tooltip auf der Oberseite">
    Tooltip oben
</button>
<button type="button" class="btn btn-secondary" data-bs-toggle="tooltip" data-bs-placement="right" title="Tooltip auf der rechten Seite">
    Tooltip rechts
</button>
<button type="button" class="btn btn-secondary" data-bs-toggle="tooltip" data-bs-placement="bottom" title="Tooltip auf der Unterseite">
    Tooltip unten
</button>
<button type="button" class="btn btn-secondary" data-bs-toggle="tooltip" data-bs-placement="left" title="Tooltip auf der linken Seite">
    Tooltip links
</button>

        
        

            
Tooltips müssen mit JavaScript initialisiert werden:

            
var tooltipTriggerList = [].slice.call(document.querySelectorAll('[data-bs-toggle="tooltip"]'))
var tooltipList = tooltipTriggerList.map(function (tooltipTriggerEl) {
    return new bootstrap.Tooltip(tooltipTriggerEl)
})

        

        
        
Popovers

        
Popovers sind ähnlich wie Tooltips, bieten aber mehr Platz für Inhalte:

        
        
<button type="button" class="btn btn-lg btn-danger" data-bs-toggle="popover" title="Popover-Titel" data-bs-content="Hier steht der Inhalt des Popovers.">
    Klick mich für Popover
</button>

        
        

            
Auch Popovers müssen initialisiert werden:

            
var popoverTriggerList = [].slice.call(document.querySelectorAll('[data-bs-toggle="popover"]'))
var popoverList = popoverTriggerList.map(function (popoverTriggerEl) {
    return new bootstrap.Popover(popoverTriggerEl)
})

Anpassung von Bootstrap

        
        
Bootstrap anpassen

        
Es gibt verschiedene Methoden, um Bootstrap an deine Bedürfnisse anzupassen:

        
        
1. Überschreiben mit eigenem CSS

        
Die einfachste Methode: Lade dein eigenes CSS nach Bootstrap:

        
        
<!-- Bootstrap CSS -->
<link href="bootstrap.min.css" rel="stylesheet">

<!-- Dein eigenes CSS (nach Bootstrap) -->
<link href="mein-stil.css" rel="stylesheet">

        
        

            
In deiner CSS-Datei kannst du Bootstrap-Stile überschreiben:

            
/* Bootstrap-Primärfarbe überschreiben */
:root {
    --bs-primary: #ff6600 !important;
    --bs-primary-rgb: 255, 102, 0 !important;
}

/* Überschreiben von Komponenten-Stilen */
.btn-primary {
    background-color: #ff6600;
    border-color: #dd5500;
}

.btn-primary:hover {
    background-color: #dd5500;
    border-color: #cc4400;
}

        

        
        
2. Bootstrap über Sass anpassen

        
Für umfangreichere Anpassungen ist Sass der bessere Weg:

        
        
// Zunächst Bootstrap-Variablen überschreiben
$primary: #ff6600;
$danger: #ff0033;
$border-radius: 0.5rem;
$border-radius-lg: 0.8rem;

// Dann Bootstrap importieren
@import "bootstrap/scss/bootstrap";

        
        

            
Die wichtigsten anpassbaren Variablen:

            
    
                
  • Farben: $primary, $secondary, $success, etc.
    • 
                
  • Schriftarten: $font-family-base, $font-size-base
    • 
                
  • Abgerundete Ecken: $border-radius
    • 
                
  • Abstände: $spacer, $grid-gutter-width
    • 
                
  • Breakpoints: $grid-breakpoints
    • 
                
  • Container-Breiten: $container-max-widths
    •

Responsive Design

        
        
Responsive Design mit Bootstrap

        
Bootstrap ist von Grund auf für responsives Design konzipiert ("Mobile First").

        
        
Grundlagen des responsiven Designs

        
    
            
  1. Viewport Meta-Tag - Stellt sicher, dass die Seite auf verschiedenen Geräten korrekt dargestellt wird
                
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
            
    2. 
            
  2. Responsive Grid-System - Verwendet Breakpoints für verschiedene Bildschirmgrößen
                
    <div class="col-12 col-md-6 col-lg-4">...</div>
    
            
    3. 
            
  3. Responsive Utilities - Kontrollieren Anzeige, Größe und Position je nach Bildschirmgröße
    4. 
        

        
        
Responsive Breakpoints in Bootstrap

        
Hier nochmals die wichtigsten Breakpoints (als Merkhilfe):

        
        
// Extra small (xs): 0px und größer (Standardwert, kein Präfix nötig)
.col-* { ... }

// Small (sm): 576px und größer
.col-sm-* { ... }

// Medium (md): 768px und größer
.col-md-* { ... }

// Large (lg): 992px und größer
.col-lg-* { ... }

// Extra large (xl): 1200px und größer
.col-xl-* { ... }

// Extra extra large (xxl): 1400px und größer
.col-xxl-* { ... }

        
        
Responsive Bilder

        
        
<!-- Bild mit responsiver Breite -->
<img src="..." class="img-fluid" alt="...">

<!-- Responsive Thumbnails -->
<img src="..." class="img-thumbnail" alt="...">

        
        
Responsive Sichtbarkeit

        
        
<!-- Auf kleinen Geräten verstecken, auf größeren anzeigen -->
<div class="d-none d-md-block">Versteckt auf Small und Extra Small</div>

<!-- Nur auf kleinen Geräten anzeigen -->
<div class="d-block d-md-none">Nur auf Small und Extra Small sichtbar</div>

<!-- Auf mittleren Geräten verstecken -->
<div class="d-md-none d-lg-block">Auf Medium unsichtbar</div>

        
        

            
Auswendig lernen: Diese Display-Klassen sind kumulativ nach oben! z.B. d-md-none versteckt ein Element auf md, lg, xl und xxl!

        

        
        
Responsive Typografie

        
        
<!-- Responsive Überschriftengröße -->
<h1 class="display-1 d-none d-md-block">Große Überschrift auf Desktop</h1>
<h1 class="display-4 d-md-none">Kleinere Überschrift auf Mobile</h1>

<!-- Responsive Text-Ausrichtung -->
<p class="text-start text-md-center text-lg-end">
    Linksbündig auf kleinen Bildschirmen, 
    zentriert auf mittleren Bildschirmen, 
    rechtsbündig auf großen Bildschirmen
</p>

        
        
Responsive Abstände (Margins/Paddings)

        
        
<!-- Unterschiedliche Margins je nach Breakpoint -->
<div class="mt-2 mt-md-3 mt-lg-5">...</div>

<!-- Unterschiedliche Paddings je nach Breakpoint -->
<div class="p-2 p-md-4 p-lg-5">...</div>

        
        
Responsive Navbar

        
Die Navbar ist standardmäßig responsiv mit dem Toggler:

        
        
<nav class="navbar navbar-expand-lg navbar-light bg-light">

            <!-- Navbarinhalte werden auf kleineren Bildschirmen ausgeblendet 
                 und können über den Toggler (Hamburger-Menü) geöffnet werden -->

        </div>
    </div>
</nav>

        
        
Praxistipps und Best Practices

        
        
Praxistipps und Best Practices

        
        
1. Mobile First Ansatz

        
Denke zuerst an mobile Geräte und skaliere dann nach oben:

        
<!-- Falsch (Desktop First) -->
<div class="col-lg-4 col-6">...</div>

<!-- Richtig (Mobile First) -->
<div class="col-6 col-lg-4">...</div>

        
        
2. Verwendung der richtigen Container

        
    
            
  • Verwende .container für responsive Websites mit fester Breite
    • 
            
  • Verwende .container-fluid für volle Breite
    • 
            
  • Verwende .container-{breakpoint} für responsive Container, die erst ab einem bestimmten Breakpoint fixiert werden
    • 
        

        
        
3. Die richtige Struktur einhalten

        

            
Immer die Hierarchie einhalten: .container → .row → .col-*

            
NIEMALS eine Row direkt unter einer anderen Row platzieren!

        

        
        
<!-- Richtig -->
<div class="container">
    <div class="row">
        <div class="col-md-6">...</div>
        <div class="col-md-6">...</div>
    </div>
    <div class="row">
        <div class="col-md-12">...</div>
    </div>
</div>

<!-- Falsch -->
<div class="container">
    <div class="row">
        <div class="row"> <!-- Row in Row! -->
            ...
        </div>
    </div>
</div>

        
        
4. Mit dem Grid-System richtig umgehen

        
    
            
  • Verwende .col ohne Zahl für automatische gleichmäßige Spaltenbreite
    • 
            
  • Nutze offset-* statt leerer Spalten
    • 
            
  • Verwende order-* für responsive Reihenfolge der Inhalte
    • 
        

        
        
5. Performance-Optimierung

        
    
            
  1. Lade nur die benötigten Bootstrap-Komponenten (über Sass)
    2. 
            
  2. Verwende den Bootstrap Bundle für JavaScript (enthält Popper.js)
    3. 
            
  3. Platziere JavaScript am Ende des <body>
    4. 
            
  4. Halte dein CSS "schlank" - vermeide unnötige Überschreibungen
    5. 
        

        
        
6. Barrierefreiheit (Accessibility)

        
    
            
  • Verwende semantisches HTML (<nav>, <article>, <section>)
    • 
            
  • Füge ARIA-Attribute zu interaktiven Elementen hinzu
    • 
            
  • Stelle ausreichenden Farbkontrast sicher
    • 
            
  • Teste deine Seite mit Screen-Readern
    • 
        

        
        
7. Responsive Bilder

        
Verwende .img-fluid für alle Bilder, die responsive sein sollen:

        
<img src="bild.jpg" class="img-fluid" alt="Responsives Bild">

        
        
8. Utility-Klassen geschickt einsetzen

        
Bootstrap 5 bietet viele nützliche Utility-Klassen, die dir CSS-Schreibarbeit ersparen:

        
    
            
  • Margins/Paddings: .m-*, .p-*
    • 
            
  • Flexbox: .d-flex, .justify-content-*, .align-items-*
    • 
            
  • Text: .text-*, .fw-*, .fs-*
    • 
            
  • Hintergrund und Farbe: .bg-*, .text-*
    • 
        

        
        
9. Häufige Fehler vermeiden

        

            
    
                
  • Vergessen des Viewport-Meta-Tags
    • 
                
  • Falsche Grid-Verschachtelung
    • 
                
  • Überschreiben von Bootstrap-Klassen statt Erweiterung
    • 
                
  • Nicht-responsive Bilder und Tabellen
    • 
                
  • Zu viele benutzerdefinierte Stile, die Bootstrap-Konsistenz brechen
    • 
            

        

        
        
10. Debugging-Tipps

        
    
            
  1. Browser-Entwicklertools nutzen, um das Layout zu inspizieren
    2. 
            
  2. Verwende .border und .bg-* temporär zum Debuggen des Grids
    3. 
            
  3. Teste auf verschiedenen Geräten oder mit Responsive-Modus im Browser
    4. 
            
  4. Verwende die Bootstrap-Dokumentation als Referenz
    5.

Responsive Breakpoints

Bootstrap verwendet Breakpoints, um responsives Verhalten zu ermöglichen. Diese Breakpoints definieren, wann sich das Layout an verschiedene Bildschirmgrößen anpasst.

Breakpoint Klassenprefix Bildschirmbreite Verwendung
Extra small col- <576px Smartphones (Portrait)
Small col-sm- ≥576px Smartphones (Landscape)
Medium col-md- ≥768px Tablets
Large col-lg- ≥992px Desktops
Extra large col-xl- ≥1200px Große Desktops
Extra extra large col-xxl- ≥1400px Sehr große Displays

Die Klassen funktionieren aufsteigend! Wenn du col-md-6 verwendest, gilt dies für Medium, Large und Extra Large Breakpoints, es sei denn, du definierst explizit andere Werte für größere Breakpoints.

Responsives Grid-Beispiel

<div class="container">
    <div class="row">
        <div class="col-12 col-md-6 col-lg-4">
            <!-- Auf Smartphones: volle Breite -->
            <!-- Auf Tablets: 6 Spalten (50%) -->
            <!-- Auf Desktops: 4 Spalten (33.33%) -->
            Inhalt...
        </div>
        <div class="col-12 col-md-6 col-lg-4">
            Inhalt...
        </div>
        <div class="col-12 col-lg-4">
            <!-- Auf Smartphones und Tablets: volle Breite -->
            <!-- Auf Desktops: 4 Spalten (33.33%) -->
            Inhalt...
        </div>
    </div>
</div>

Container-Arten

Bootstrap bietet verschiedene Container-Arten:

Klasse Beschreibung
.container Responsive Container mit fester Breite, die sich an Breakpoints anpasst
.container-fluid Container mit voller Breite (100%) auf allen Breakpoints
.container-{breakpoint} 100% breit bis zum angegebenen Breakpoint, dann wie .container
z.B.: .container-sm, .container-md

IMMER WICHTIG: Container → Row → Columns. Diese Hierarchie muss eingehalten werden!

Zusätzliche Grid-Funktionen

Spalten-Versatz (Offsetting)

<div class="row">
    <div class="col-md-4">Inhalt</div>
    <div class="col-md-4 offset-md-4">Versetzt um 4 Spalten</div>
</div>

Spalten-Reihenfolge (Order)

<div class="row">
    <div class="col-md-4 order-md-2">Wird als zweites angezeigt</div>
    <div class="col-md-4 order-md-3">Wird als drittes angezeigt</div>
    <div class="col-md-4 order-md-1">Wird als erstes angezeigt</div>
</div>

Verschachtelte Grids (Nesting)

<div class="row">
    <div class="col-sm-9">
        Äußere Spalte
        <div class="row">
            <div class="col-8">Verschachtelte Spalte</div>
            <div class="col-4">Verschachtelte Spalte</div>
        </div>
    </div>
</div>
